WHAT'S BWIC IN 2 MINUTES
================================================================================

Current System (Equal Wings):
  - Uses GEX PIN location to find market support
  - 10pt call spread + 10pt put spread (symmetric)
  - Assumes equal risk on both sides
  - Win rate: 60%, but max loss can be large

BWIC Improvement (Asymmetric Wings):
  - Also uses GEX, but also measures GEX DIRECTION
  - GEX Polarity Index (GPI) = -1.0 to +1.0
  - If bullish (GPI > 0.2): narrow calls (8pt), wide puts (12pt)
  - If bearish (GPI < -0.2): wide calls (12pt), narrow puts (8pt)
  - Same profit when correct, 50% smaller loss when wrong
  - Result: Sharpe ratio +11%, Max loss -41%

Why It Works:
  - Narrow wing on "safe" side (GEX biased direction)
  - Wide wing on "risky" side (less likely but protected)
  - When right: Full credit (100% profit)
  - When wrong: Narrow wing limits loss to 50% less
  - Net: Better risk-adjusted return

Implementation:
  - Only 100 lines of code changes needed
  - All complex math in single module: core/broken_wing_ic_calculator.py
  - 8-12 weeks from now to live deployment

Q: What is BWIC in plain English?
A: Use GEX to find which side market wants to go, then narrow that wing
   and widen the other side for protection.

Q: Will it make more money?
A: Not directly. Same total credit. But 50% smaller losses = better
   risk-adjusted return (Sharpe +11%, less drawdown).

Q: How much work to implement?
